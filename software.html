In the middle of 2021, I developed a chat program according to an exercise assignment of a lecture, with which two people in the same network could write with each other via the console window.


After successfully completing the task, I then redesigned the program so that more than two people could communicate with each other. I changed from a 1:1 direct communication of the programs to a server/client structure. Thus it was possible to select whether the corresponding instance of the program should be the server or a client.


The server could also be configured on different ports, so there could be multiple servers on the same network and on the same machine. A client could then log in with the corresponding IP address and its own user name (if not already assigned).


Since my focus at the time was only on the communication itself and the backend, the appearance of the program remained in the console window.
