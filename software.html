In the middle of 2021, I developed a chat program according to an exercise assignment of a lecture, with which two people in the same network could write with each other via the console window.


After successfully completing the task, I then redesigned the program so that more than two people could communicate with each other. I changed from a 1:1 direct communication of the programs to a server/client structure. Thus it was possible to select whether the corresponding instance of the program should be the server or a client.


The server could also be configured on different ports, so there could be multiple servers on the same network and on the same machine. A client could then log in with the corresponding IP address and its own user name (if not already assigned).


Since my focus at the time was only on the communication itself and the backend, the appearance of the program remained in the console window.

In the practical part of the database lecture 2021/22 my team and I developed a web application with which users can either offer or book rides and write ratings.


The focus of the task was on database access using sql and the basic security of database queries. On the main page you could see the offered rides as well as the booked rides. On the corresponding detail page (picture) the respective details and ratings were displayed. A new ride could be created on a similar page with a corresponding form.


The backend was programmed in Python using the Flask framework for the webserver and the database was accessed via DB2-SQL. During the project I worked on the frontend as well as on the backend.

As practical course of my studies, I participated in the project "ChuuChuUDP" of the department of distributed systems in 2021/22. Our team had the task to automate the operation of a model railway system in such a way that the user could click on target points on the track and the train would navigate there on its own, as well as select ready-made routes.


The corresponding selection and direct control was possible via a web page, which then transmitted the entered commands via HTTP to the backend programmed in Go. This then took control of the corresponding trains and steered them to the selected destinations. The communication with the model railroad was realized via UDP between the backend and the Z21 control station by ROCO, which then forwarded the commands to the trains via DCC.


My activities during the project were generally located in the whole backend, but focused on realizing the communication between backend and Z21 and designing a graph structure to be able to use the track layout in the digital twin.
